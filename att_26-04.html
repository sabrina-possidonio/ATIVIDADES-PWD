Respostas - Atividades de Desenvolvimento Web

Atividade HTML:

  1. O que é HTML e qual é o seu papel na criação de páginas web?
    HTML é a linguagem padrão para criar e estruturar páginas web. Seu papel é definir a estrutura e o conteúdo de uma página, permitindo a apresentação de informações de forma organizada e acessível na web.
  2. Qual é a função de uma tag em HTML? Dê um exemplo de uma tag.
    Uma tag em HTML é usada para marcar e identificar diferentes partes de um documento web. Por exemplo, a tag <p> é usada para definir um parágrafo.
  3. Explique o conceito de elemento em HTML com um exemplo.
    Um elemento HTML é formado por uma tag de abertura, conteúdo e, em muitos casos, uma tag de fechamento. Por exemplo: <p>Este é um parágrafo</p>.
  4. O que são atributos em HTML e como são utilizados? Dê um exemplo de atributo.
    Os atributos em HTML fornecem informações adicionais sobre um elemento e são definidos dentro das tags de abertura. Um exemplo de atributo é o "href" em um elemento <a>, que define o link de destino. Por exemplo: <a href="https://www.example.com">Link Exemplo</a>.
  5. Qual é a versão mais recente do HTML e quais são alguns dos recursos avançados suportados por ela?
    A versão mais recente do HTML é o HTML5, que suporta recursos avançados como elementos de vídeo e áudio nativos, armazenamento local, gráficos vetoriais, entre outros.

Atividades CSS:

  1. Qual é o propósito principal do CSS?
    O propósito principal do CSS é estilizar elementos HTML e controlar sua apresentação visual em uma página da web, separando a estrutura do conteúdo HTML de sua aparência.
  2. Quais são alguns dos principais recursos e conceitos do CSS mencionados no texto?
    Alguns dos principais recursos e conceitos do CSS incluem seletores, propriedades CSS (como cor, tamanho e espaçamento), classes e IDs, Box Model, Flexbox e Grid.
  3. Quais são alguns dos desafios enfrentados pelos desenvolvedores ao trabalhar com CSS e como podem ser mitigados?
    Os desenvolvedores enfrentam desafios como compatibilidade entre navegadores e desempenho. Esses desafios podem ser mitigados testando em diferentes navegadores e seguindo práticas para melhorar o desempenho, como minificar e agrupar arquivos CSS.
  4. O que é o Box Model no contexto do CSS e por que é importante entender?
    O Box Model é um conceito fundamental do CSS que descreve como os elementos HTML são renderizados na página, incluindo margens, bordas, preenchimento e conteúdo. É importante entender o Box Model para controlar o layout e o espaçamento dos elementos de uma página web.
  5. Quais são algumas das técnicas de layout avançadas do CSS mencionadas no texto?
    Algumas das técnicas de layout avançadas do CSS mencionadas incluem Flexbox e Grid, que permitem criar layouts flexíveis e responsivos.

Git e GitHub:

  1. Qual é a diferença entre o Git e o GitHub?
    O Git é um sistema de controle de versão distribuído, enquanto o GitHub é uma plataforma de hospedagem de código que utiliza o Git para facilitar a colaboração entre desenvolvedores.
  2. Quais são os principais conceitos do Git mencionados no texto e como eles são importantes para o controle de versão?
    Alguns dos principais conceitos do Git incluem repositórios, branches, merge e conflitos de merge. Esses conceitos são importantes para rastrear e gerenciar alterações em projetos de software ao longo do tempo, garantindo a integridade do código e facilitando a colaboração entre desenvolvedores.
  3. Quais são os desafios enfrentados pelos desenvolvedores ao trabalhar com Git e GitHub e como podem ser mitigados?
    Os desafios incluem resolução de conflitos de merge e colaboração em projetos distribuídos. Eles podem ser mitigados seguindo práticas recomendadas, como manter commits pequenos e frequentes, revisar cuidadosamente as alterações antes de mesclar e resolver conflitos de merge de forma adequada.
  4. Como o Git e o GitHub revolucionaram a forma como os desenvolvedores colaboram e gerenciam projetos de software?
    O Git e o GitHub proporcionaram uma maneira eficiente e distribuída de rastrear e gerenciar alterações no código fonte, permitindo que os desenvolvedores colaborem em tempo real, revisem o código uns dos outros e gerenciem projetos de software de maneira mais eficaz, independentemente da localização geográfica.
  5. Por que é importante seguir práticas recomendadas ao trabalhar com Git e GitHub?
    É importante seguir práticas recomendadas para garantir uma colaboração eficaz e um gerenciamento de projeto adequado. Isso inclui manter commits pequenos e frequentes, revisar cuidadosamente as alterações antes de mesclar e resolver conflitos de merge de forma adequada, garantindo a integridade do código e facilitando a colaboração entre desenvolvedores.
